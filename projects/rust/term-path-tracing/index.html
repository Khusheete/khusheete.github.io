Terminal Path Tracing

This project builds off of my Terminal Renderer to render an image in the terminal.

It currently only contains an unoptimized CPU path tracer wich does not support advanced materials. This implementation is a prototype I made to figure out how to architecture one. This project's goal is to create a path tracer on the graphics card using the Vulkan API, and have it run in real time in the terminal. A full light simulation should be possible as the rendering surface is quite small, and the terminal renderer allows to draw images quite fast as long as images are quite close from one frames to the next.

To achieve that goal, I've been reading about Vulkan on the khronos specification page and in the vulkan.org documentation. I've also been experimenting with ash, a library implementing Vulkan bindings for the rust programming language.


The source code is available on github under the MIT license.
